# Kid Games — Design

วันที่: 2026-06-28
สถานะ: อนุมัติดีไซน์แล้ว รอเขียนแผน implementation

## 1. ภาพรวมและเป้าหมาย

ชุดเกมการเรียนรู้สำหรับเด็กอายุ 4-6 ขวบ ฝึกทายคำศัพท์ ฝึกอ่าน และฝึกคำนวณ
ใช้งานเป็นเว็บ เปิดเล่นได้ทันทีบนแท็บเล็ต/คอมพิวเตอร์ ไม่ต้องติดตั้ง

**ผู้ใช้:** ลูกของเจ้าของโปรเจกต์คนเดียว ไม่มีระบบสมาชิก/ล็อกอิน
**การเก็บข้อมูล:** เก็บความคืบหน้า (ดาวสูงสุดของแต่ละเกม) ใน `localStorage` ของ browser เท่านั้น ไม่มี backend

## 2. ขอบเขต

### รอบแรก (MVP) — สร้างจริงในแผนนี้
เกมเลือกตอบ 4 ตัวเลือก 3 เกม ที่ใช้ `QuizEngine` ร่วมกัน:

1. **เกมเลข** — สุ่มเลข 1-10 แสดงตัวเลขใหญ่ → เลือกคำสะกดภาษาอังกฤษ (ONE, TWO, …)
2. **เกมสี** — สุ่มสีจากชุดสีพื้นฐาน แสดงช่องสี → เลือกคำศัพท์สี (RED, BLUE, …)
3. **เกมบวกเลข** — สุ่ม a + b โดยผลรวม ≤ 10 แสดง "2 + 3 = ?" → เลือกตัวเลขคำตอบ

### รอบถัดไป (ออกแบบรองรับไว้ แต่ยังไม่สร้างในแผนนี้)
4. **เกมเติมคำ (Fill-in)** — แสดงเลขเป็นภาพ (เช่น 🍎🍎🍎) มีช่องว่าง 1 ช่อง และคลังคำ ~10 คำ
   ให้แตะคำที่ถูกเด้งลงช่อง ตั้งค่าได้ (จำนวนคำในคลัง, โชว์คำใบ้ 1 ตัวหรือไม่)
5. **เกมอ่านจับใจความภาษาไทย** — ยังไม่ออกแบบในรอบนี้

`QuizEngine` ต้องออกแบบให้รองรับ "โหมดการแสดงผล" ได้หลายแบบ (ปุ่มเลือกตอบ / คลังคำเติมช่อง)
เพื่อให้เกมที่ 4 ต่อยอดได้โดยไม่ต้องแก้ตรรกะหลัก

## 3. สถาปัตยกรรม

React + Vite + TypeScript — SPA ฝั่ง client ล้วน

```
kid-games/
├── หน้าหลัก (Home)       เลือกเกมจากการ์ด 3 ใบ (ไอคอนใหญ่ + ชื่อไทย)
├── หน้าเล่นเกม (Play)     ใช้ QuizEngine ร่วมกัน
├── หน้าสรุปผล (Result)   ดาวรางวัล + เล่นอีกครั้ง / กลับหน้าหลัก
└── localStorage         ดาวสูงสุดของแต่ละเกม
```

### โครงสร้างหน่วยงาน (units)

- **`QuizEngine`** — คอมโพเนนต์/ฮุกกลาง: รับ `GameDefinition` แล้วจัดการ flow ทั้งหมด
  (สุ่มโจทย์ → แสดงผ่านโหมดที่กำหนด → รับคำตอบ → เช็ค → เล่นเสียง → นับคะแนน → เดินหน้าจนครบ 10 ข้อ → ส่งผลไปหน้าสรุป)
- **`GameDefinition`** — ข้อมูลเฉพาะของแต่ละเกม (ดูสัญญาด้านล่าง) แยกไฟล์ละเกม
- **`PromptRenderer`** — แสดงโจทย์ตามชนิด (ตัวเลขใหญ่ / ช่องสี / สมการบวก)
- **`ChoiceButtons`** — โหมดแสดงผลแบบปุ่ม 4 ปุ่ม (โหมดเดียวที่ใช้ใน MVP)
- **`audio`** — ห่อ Web Speech API (พูดคำศัพท์) + Web Audio (โทนถูก/ผิด)
- **`storage`** — อ่าน/เขียนดาวสูงสุดใน localStorage
- **`scoring`** — แปลงจำนวนข้อถูกเป็นจำนวนดาว

### สัญญา (interfaces)

```ts
type GameDefinition = {
  id: string                       // เช่น 'numbers'
  title: string                    // ชื่อไทยบนการ์ด
  icon: string                     // อิโมจิ/ไอคอน
  generateQuestion: () => Question // สุ่มโจทย์ 1 ข้อ
}

type Question = {
  prompt: Prompt                   // ข้อมูลโจทย์ (ดูชนิดด้านล่าง)
  speakText: string                // ข้อความให้ TTS อ่านตอนตอบถูก เช่น "five"
  choices: Choice[]                // 4 ตัวเลือก
}

type Choice = { id: string; label: string; correct: boolean }

type Prompt =
  | { kind: 'number'; value: number }            // แสดงตัวเลขใหญ่
  | { kind: 'color'; hex: string; name: string } // แสดงช่องสี
  | { kind: 'math'; a: number; b: number }        // แสดง "a + b = ?"
```

## 4. ตัวสร้างโจทย์ (รอบแรก)

- **เกมเลข:** สุ่ม `value` 1-10 → `prompt = {kind:'number', value}` → คำตอบถูก = คำอังกฤษของ value, ตัวลวง 3 ตัวสุ่มจากคำอื่นไม่ซ้ำ, `speakText` = คำอังกฤษ
- **เกมสี:** สุ่มสีจาก palette (แดง ส้ม เหลือง เขียว ฟ้า น้ำเงิน ม่วง ชมพู ดำ ขาว น้ำตาล) → `prompt = {kind:'color'}` → คำตอบถูก = ชื่อสีอังกฤษ, ตัวลวง 3 ตัวสุ่มจากสีอื่น, `speakText` = ชื่อสี
- **เกมบวกเลข:** สุ่ม `a`, `b` (≥1) ที่ `a+b ≤ 10` → `prompt = {kind:'math', a, b}` → คำตอบถูก = ผลรวม, ตัวลวง 3 ตัวเป็นตัวเลขใกล้เคียงไม่ซ้ำและอยู่ในช่วงสมเหตุผล, `speakText` = ผลรวม

**กติกาตัวลวง:** ทุกข้อมีคำตอบถูก 1 ตัว + ตัวลวง 3 ตัวที่ไม่ซ้ำคำตอบและไม่ซ้ำกันเอง รวม 4 ตัว แล้วสับลำดับ

## 5. การเล่น เสียง และรางวัล

**Flow 1 ข้อ:** แสดงโจทย์ + อ่านโจทย์/คำใบ้ → เด็กแตะตัวเลือก
- ถูก: ขอบเขียว + เสียงชม + TTS อ่าน `speakText` → หน่วงสั้นๆ → ข้อถัดไป
- ผิด: ขอบแดง + ไฮไลต์ตัวเลือกที่ถูก → หน่วง → ข้อถัดไป
- นับคะแนนจากการตอบถูกครั้งแรกของแต่ละข้อ

**รอบการเล่น:** 10 ข้อต่อรอบ มีแถบความคืบหน้าให้เห็นว่าเหลือกี่ข้อ

**เสียง:**
- พูดคำศัพท์/คำตอบ: Web Speech API (`speechSynthesis`) — มีในตัว browser ไม่ต้องมีไฟล์เสียง
- เสียงตอบรับถูก/ผิด: Web Audio API สร้างโทนสั้นๆ (ไม่ต้องมีไฟล์เสียง)
- ถ้า browser ไม่รองรับเสียง เกมยังเล่นได้ปกติ (เสียงเป็นส่วนเสริม ไม่บล็อกการเล่น)

**ดาว (ตอนจบรอบ):** ตอบถูก 8-10 ข้อ = ⭐⭐⭐, 5-7 = ⭐⭐, 1-4 = ⭐, 0 = ไม่มีดาว
เก็บดาวสูงสุดของแต่ละเกมไว้ใน localStorage แสดงบนการ์ดหน้าหลัก

## 6. หน้าตา (UX เด็ก 4-6)

ปุ่มใหญ่กดง่าย (touch target ใหญ่) สีสดใส มุมมน ตัวอักษรใหญ่ ข้อความน้อย เน้นไอคอน
เมนู/ข้อความระบบเป็นภาษาไทย ส่วนเนื้อหาเกม (คำศัพท์อังกฤษ) เป็นภาษาอังกฤษตามวัตถุประสงค์

## 7. การทดสอบ

Vitest ทดสอบตรรกะล้วน (ไม่พึ่ง DOM):
- ตัวสร้างโจทย์แต่ละเกม: คำตอบที่ถูกอยู่ในตัวเลือกเสมอ, มีตัวเลือก 4 ตัวไม่ซ้ำกัน, ค่าอยู่ในช่วงที่กำหนด (เลข 1-10, ผลบวก ≤ 10)
- `scoring`: แปลงจำนวนข้อถูกเป็นจำนวนดาวถูกต้องตามเกณฑ์
- `storage`: เขียนแล้วอ่านดาวสูงสุดกลับมาได้ และเก็บค่าสูงสุดจริง (ไม่ทับด้วยค่าที่ต่ำกว่า)

## 9. สิ่งที่ตัดออกโดยตั้งใจ (YAGNI)

- ไม่มี backend / ฐานข้อมูล / ล็อกอิน / ระบบหลายผู้ใช้
- ไม่มีไฟล์เสียงภายนอก (ใช้ API ในตัว browser)
- รอบแรกไม่มีหน้า Settings (จะมาพร้อมเกมเติมคำในรอบถัดไป)
- รอบแรกไม่มีระดับความยาก (สุ่มช่วงคงที่)
